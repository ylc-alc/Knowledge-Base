# Knowledge Summary
A practical technical solution for masking sensitive data during live demos using a custom 'Recording On' skill in Claude Code to replace identifiers with fake names and addresses.

# Extracted Details
- Privacy Risk: Demonstrating live AI workflows often accidentally reveals sensitive data like client addresses, boss names, or school locations. [cite: 2]
- "Recording On" Skill: A custom function for Claude Code that, when activated, automatically swaps all identifiable information with fake data (pseudonyms/fictional addresses). [cite: 2]
- Logic Preservation: The AI maintains the underlying logic and structure of the workflow while obfuscating the private content. [cite: 2]
- Context Switching: This demonstrates that `claude.md` files can be used for "scenario-based modes" (like a privacy mode) rather than just static rules. [cite: 2]

# Key Takeaways
- Use `claude.md` to define distinct operational modes, such as a 'Privacy Mode' for public demonstrations. [cite: 2]
# Actionable Tips
- Create a 'Recording On' trigger that instructs your AI agent to use placeholders for all PII (Personally Identifiable Information) during screen shares. [cite: 2]
- Implement a 'Recording Off' command to revert to normal data processing once the demo is complete. [cite: 2]
